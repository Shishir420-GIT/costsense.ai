### Example: Dashboard Summary Query Flow

```
1. API Request
   GET /api/v1/dashboard/summary

2. Router Handler
   dashboard_repo = DashboardRepository(db)
   data = dashboard_repo.get_dashboard_summary()

3. Repository Layer
   def get_dashboard_summary():
       # Check Redis cache
       cached = redis.get("dashboard:summary")
       if cached:
           return json.loads(cached)

       # Query database
       latest_metric = db.query(DashboardMetric)\
           .order_by(desc(DashboardMetric.date))\
           .first()

       # Aggregate costs
       daily_costs = db.query(
           AzureCost.date,
           func.sum(AzureCost.cost)
       ).group_by(AzureCost.date)\
        .order_by(desc(AzureCost.date))\
        .limit(30)\
        .all()

       # Top services
       top_services = db.query(
           AzureCost.service_name,
           func.sum(AzureCost.cost)
       ).group_by(AzureCost.service_name)\
        .order_by(desc(func.sum(AzureCost.cost)))\
        .limit(5)\
        .all()

       # Build response
       result = {
           "total_monthly_cost": latest_metric.total_monthly_cost,
           "monthly_change_percent": latest_metric.monthly_change_percent,
           "daily_costs": [...],
           "top_services": [...]
       }

       # Cache for 60 seconds
       redis.setex("dashboard:summary", 60, json.dumps(result))

       return result

4. Response
   {
     "total_monthly_cost": 6225.21,
     "monthly_change_percent": 2.2,
     "daily_costs": [...],
     "top_services": [...]
   }
```

## Development vs Production

### Development Mode
```
- SQLite database (file-based, easy setup)
- Redis optional (works without it)
- Ollama optional (fallback analysis)
- Mock data seeding script
- Hot reload enabled
- Detailed error messages
```

### Production Mode (Future)
```
- PostgreSQL database (scalable, concurrent)
- Redis required (performance critical)
- Ollama or cloud LLM (better AI responses)
- Real Azure data integration
- Optimized builds
- Generic error messages (security)
- Monitoring & logging
- Backup strategies
```
